# DeFi资产展示服务 - 对标DeBank功能

## 项目概述
开发一个Go语言服务，用于展示用户在DeFi协议中的资产持仓情况。

## 核心功能
1. **服务A集成** - 调用外部服务获取有balance概念的协议资产（实时查询，不存储）
2. **服务B集成** - 调用外部服务获取无balance概念的协议仓位数据（存入MySQL，Redis缓存）
3. **协议元数据同步** - 从DeBank网页抓取协议信息，定时同步
4. **Redis队列处理** - 接收服务B的实时仓位更新

## 系统架构
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  前端/API调用方  │───▶│   Go API网关    │───▶│   服务A调用     │
│                 │    │                 │    │ (实时balance)   │
└─────────────────┘    └─────────────────┘    └─────────────────┘
                              │
                              ├─▶│   服务B调用     │───▶ MySQL存储
                              │  │ (协议API数据)   │
                              │  └─────────────────┘
                              │           │
                              │           └─▶ Redis缓存 (TTL: 10分钟)
                              │
                              ├─▶│ 协议元数据同步  │───▶ DeBank网页抓取
                              │  └─────────────────┘
                              │
                              └─▶│ Redis队列监听  │◀─── 服务B更新推送
                                 └─────────────────┘
```

## 技术栈
- **后端**: Go 1.21+
- **数据库**: MySQL 8.0+
- **缓存**: Redis 7.0+
- **队列**: Redis Streams/PubSub
- **HTTP客户端**: Go标准库/net/http
- **网页抓取**: Colly/GoQuery
- **定时任务**: cron库

## 数据流
1. **实时balance查询** → 服务A → 直接返回
2. **协议仓位查询** → 检查Redis → 命中则返回 → 未命中则调用服务B → 存入MySQL和Redis
3. **协议元数据** → 定时从DeBank抓取 → 存入MySQL
4. **实时更新** → 服务B推送 → Redis队列 → 更新MySQL和Redis

## 预计完成时间: 6-8小时
