# Go服务开发任务指令

## 项目背景
开发一个对标DeBank功能的DeFi资产展示服务，需要实现完整的Go后端服务。

## 技术栈要求
- Go 1.21+
- MySQL 8.0+ (使用GORM或sqlx)
- Redis 7.0+ (使用go-redis)
- Web框架: Gin或Echo（推荐Gin）
- 配置管理: Viper
- 日志: Zap或Logrus
- HTTP客户端: 标准库或resty

## 核心功能模块

### 1. API网关模块
- RESTful API设计
- 路由管理
- 中间件（日志、认证、限流）
- 请求/响应处理

### 2. 数据库模块
- MySQL连接管理
- 数据模型定义
- 数据访问层（DAO）
- 迁移脚本

### 3. Redis模块
- Redis连接管理
- 缓存操作封装
- 队列操作（Streams/PubSub）
- 缓存策略实现

### 4. 外部服务集成模块
- 服务A客户端（实时balance查询）
- 服务B客户端（协议API调用）
- HTTP客户端封装
- 错误重试机制

### 5. 业务逻辑模块
- 资产查询聚合逻辑
- 缓存逻辑（先查Redis，未命中查DB）
- 数据同步逻辑
- 队列处理逻辑

### 6. 定时任务模块
- 协议元数据同步任务
- 缓存清理任务
- 数据统计任务

## 代码质量要求
- 遵循Go代码规范
- 完善的错误处理
- 单元测试覆盖核心逻辑
- 清晰的日志输出
- 配置化设计

## 输出要求
创建完整的、可运行的Go项目代码。
