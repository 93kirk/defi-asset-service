# 架构设计任务指令

## 项目背景
开发一个对标DeBank功能的DeFi资产展示服务，需要调用两个外部服务：
1. 服务A：处理有balance概念的协议（直接调用合约查询余额）
2. 服务B：处理无balance概念的协议（通过协议API查询）

## 核心需求
1. **服务A集成**：实时调用，不存储数据
2. **服务B集成**：调用后数据存入MySQL，查询时缓存到Redis（TTL可配置）
3. **协议元数据同步**：从DeBank网页抓取，每天定时同步
4. **Redis队列**：接收服务B的实时仓位更新

## 设计约束
- 使用Go语言开发
- MySQL 8.0+ 数据库
- Redis 7.0+ 缓存和队列
- 需要支持高并发查询
- 数据一致性要求高

## 设计重点
1. **性能优化**：balance查询要尽可能实时，协议数据要有合理缓存
2. **数据一致性**：服务B的数据更新要通过队列保证一致性
3. **可扩展性**：协议数量可能增长，系统要能水平扩展
4. **监控告警**：关键操作要有监控和日志
