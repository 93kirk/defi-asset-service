# DeFi资产展示服务 - 数据同步系统项目总结

## 项目概述

本项目实现了DeFi资产展示服务的数据同步系统，对标DeBank功能，负责从DeBank网页抓取协议元数据并定时同步到本地数据库。系统基于Go语言开发，采用模块化设计，具备高可用、高性能和易扩展的特点。

## 同步机制

### 全量同步
- **触发**: 每天凌晨2点
- **流程**: 抓取所有协议 → 对比数据库 → 更新变更 → 清理过时数据
- **特点**: 保证数据完整性

### 增量同步
- **触发**: 每5分钟检查
- **流程**: 检查需要同步的协议 → 抓取协议详情 → 更新变更
- **特点**: 高效、实时

### 错误处理
- **重试策略**: 指数退避，最多3次
- **熔断机制**: 连续失败时暂停
- **错误分类**: 可重试错误和不可重试错误
- **状态跟踪**: 记录错误次数和时间

## 使用说明

### 快速开始
```bash
# 使用Docker Compose
docker-compose up -d

# 或手动部署
make build
make migrate
make run
```

### 配置说明
```yaml
# 主要配置项
sync:
  protocol_metadata:
    enabled: true
    schedule: "0 0 2 * * *"  # 每天凌晨2点
    batch_size: 50
    concurrency: 5
```

### API接口
```
GET  /health                    # 健康检查
GET  /metrics                   # 监控指标
GET  /api/v1/sync/status        # 同步状态
POST /api/v1/sync/trigger       # 手动触发同步
GET  /api/v1/protocols          # 协议列表
```
